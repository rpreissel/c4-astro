## Schnellstart

### Workflow: Dokumentationsseite mit UML-Diagramm

1. **Dokumentationsseite erstellen** → [Guide](./dokumentationsseite-erstellen/)
   - MDX-Datei in `astro_starlight/src/content/docs/` anlegen
   - Frontmatter und Inhalt schreiben

2. **UML-Diagramm erstellen** → [Guide](./uml-diagramm-erstellen/)
   - Datei in `uml/` anlegen
   - PlantUML-Syntax schreiben

3. **UML-Diagramm einbinden** → [Guide](./uml-einbinden/)
   - PlantUML-Komponente importieren
   - Diagramm einbetten

4. **Zur Sidebar hinzufügen** → [Guide](./sidebar-anpassen/)
   - `astro.config.mjs` bearbeiten
   - Seite zur Navigation hinzufügen

### Workflow: C4-Architekturmodell erweitern

1. **Bestehendes Modell erweitern** → [Guide](./c4-modell-erweitern/)
   - Neue Elemente, Container oder Komponenten hinzufügen
   - Beziehungen definieren
   - Neue Views erstellen

2. **Dokumentationsseite aktualisieren**
   - Bestehende View-Komponente nutzen
   - Neue View-IDs einbinden

3. **Optional: Zur Sidebar hinzufügen** → [Guide](./sidebar-anpassen/)
   - Neue Seiten zur Navigation hinzufügen

### Workflow: Komplett neues C4-Projekt

1. **Neues C4-Projekt anlegen** → [Guide](./c4-neues-projekt/)
   - C4-Projektordner erstellen
   - Spezifikation, Modell und Views definieren
   - Eigene Astro-Komponente erstellen

2. **Dokumentationsseite erstellen** → [Guide](./dokumentationsseite-erstellen/)
   - MDX-Datei anlegen
   - C4View-Komponente importieren und nutzen

3. **Zur Sidebar hinzufügen** → [Guide](./sidebar-anpassen/)
   - Navigation erweitern

## Technologie-Stack

Diese Dokumentationsplattform nutzt:

- **Astro Starlight** - Dokumentations-Framework
- **PlantUML** - UML-Diagramme
- **LikeC4** - C4-Modell-Diagramme
- **MDX** - Markdown mit Komponenten
- **React** - Interaktive Komponenten
